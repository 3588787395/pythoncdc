## 使用指南

### 如何阅读ADR
1. 从本README的索引表找到感兴趣的决策
2. 点击链接查看完整的决策记录
3. 每个ADR遵循标准格式：背景→选项→决策→理由→后果→相关文档

### 如何添加新ADR
1. 在 `docs/architecture_decisions/` 目录下创建 `ADR-XXX.md` 文件
2. 按照标准模板格式编写内容
3. 更新本文件的索引表
4. 提交PR并经过团队评审

### ADR状态说明
- **✅ 已采纳**: 当前生效的架构决策
- **🔄 待定**: 正在评估或计划中的决策
- **❌ 已废弃**: 被新决策替代的旧决策

## 架构原则总览

本项目遵循以下核心架构原则：

1. **识别与生成分离**: 区域分析阶段完成所有语义判断，AST生成阶段只做纯翻译
2. **算法驱动而非启发式**: 使用基于编译器理论的确定性算法，避免特殊case处理
3. **结构特征检测优先**: 基于控制流结构和支配关系判断语义，不依赖操作码名称
4. **单一职责原则**: 每种区域类型有唯一的处理方法，每个方法职责明确
5. **功能验证优先**: 以字节码一致性为核心验证标准，而非仅检查语法正确性
